# 结论先行：技术栈选型 & 核心架构
**目标**：手机App，本地服务器**多Profile常驻在线**、角色卡片切换、随时对话、后台保活、**不用SSH**。
**最优方案**：前端 **Flutter** + 后端 **Go** + 通信 **WebSocket** + 多Agent管理 **Hermes**。
**架构**：手机App <--> 长连接网关 <--> 本地多Agent集群

## 一、为什么直接Pass SSH
- SSH 是**命令行一次性连接**，无法做到**常驻在线、实时感知、多角色同时在线**。
- 无法做卡片UI、状态显示、实时消息推送。
- 手机网络波动、后台切App就断连，**无法保活**。

**核心替代**：**WebSocket 长连接**，永久在线、低延迟、手机随时唤醒。

## 二、最终定型技术栈（最贴合你的需求）
### 1. 手机端 App（唯一推荐）
**Flutter**
- 一套代码打包 **安卓 + iOS**
- 完美实现：**Metro风格卡片列表、多角色并排、点谁聊谁**
- 后台保活、常驻后台、断线自动重连
- UI流畅、轻量化、低耗电

### 2. 服务端（你的机器，跑多个Profile）
- 语言：**Golang**
- 核心：自建 **WebSocket 服务**
- 能力：
  - 同时挂载 **N个 Agent Profile**
  - 每个角色独立会话、独立人设、独立记忆
  - 全部**常驻在线**，不用重新启动
  - 手机随时连上，秒切对话

### 3. 多智能体内核
直接对接你现在用的：**Hermes**
- Hermes 负责：多角色隔离、人设、记忆、Agent调度
- 你的后端做：**连接转发、在线管理、卡片状态、手机端推送**

### 4. 通信方案
- **WebSocket 长连接**（主力）
- 断网自动重连、心跳保活
- 手机切后台、锁屏依然**保持在线**
- 完全替代SSH，体验丝滑

## 三、产品形态（你想要的样子）
1. App首页：一排**卡片式角色**（每个Profile一个）
2. 所有角色**同时在线常驻**
3. 点击任意卡片 → 切入和该Agent对话
4. 其他角色卡片留在侧边/底部，随时切换
5. 服务端机器：所有Agent**永不掉线、后台常驻**
6. 手机随时随地打开就能聊，不用重新连接

## 四、为什么这套是最优解
1. **真正多智能体同时在线**，不是切换登录
2. 手机**无需SSH**，公网/内网都能连
3. 完美**保活、后台驻留、自动重连**
4. Flutter做UI最适配你要的**卡片极简风格**
5. Go后端轻量、高性能、扛多并发Agent
6. 直接复用你现有 **Hermes 多Profile**

```
手机 Flutter App
   ↓ 长连接WebSocket
Go 网关服务
   ↓
Hermes 多Agent内核
→ 多个Profile 常驻并发运行
```
